    \section{Tooling, Documentation, and
Community}\label{tooling-documentation-and-community}

    \subsection{Tooling}\label{tooling}

The two most commons tools for coding in Julia are:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\itemsep1pt\parskip0pt\parsep0pt
\item
  The Julia REPL (the terminal)
\item
  Juno, the IDE in Atom
\item
  Jupyter notebooks
\end{enumerate}

Other solutions exist (SublimeText, Emacs, etc.). The general mantra is:

\begin{quote}
Jupyter notebooks are for code, Juno is for data. (Mike Innes, developer
of Juno)
\end{quote}

Jupyter notebooks do not easily show code from multiple files, do not
have a debugging GUI, and it doesn't have as easy access to a console,
making it less friendly for developing large projects (like packages).
However, Jupyter notebooks store their outputs in a convenient form,
which is good for saving and sharing results. I suggest that you learn
both and use the right tool for the right job.

    \subsubsection{Jupyter Notebooks with no
Install}\label{jupyter-notebooks-with-no-install}

Go to www.juliabox.com and sign in to start using Julia on a cloud-based
Jupyter notebook. (Note: you may need to use \texttt{Pkg.update()} to
update your local package database (``METADATA'') on first use).

To setup the workshop notebooks in JuliaBox, click on the ``Sync'' tab
and under ``Git Repositories'' paste in
https://github.com/UCIDataScienceInitiative/IntroToJulia for the ``Git
Clone URL'', and click the +. Upon refreshing the homepage you should
see the IntroToJulia folder appear. Click into this folder, and click
into the Notebooks folder.

    \subsubsection{Installing Julia}\label{installing-julia}

Unless you're a serious hacker, I recommend installing Julia by
downloading one of the binaries from http://julialang.org/downloads/.
There are ready-to-use binaries for most operating systems. Just
download, run the installer, and use Julia.

If you want to build from the source, the repository along with the
build instructions are at https://github.com/JuliaLang/julia.

    \subsection{Community}\label{community}

Julia is changing fast: there are updates every few months which bring
in loads of new features, packages are updating all of the time, and the
documentation, while good, is not entirely complete. The best fix for
these problems is to be engaged in the Julia community.

\paragraph{I would highly recommend finding a good online community
while learning
Julia}\label{i-would-highly-recommend-finding-a-good-online-community-while-learning-julia}

The main Julia community resources are:

\begin{itemize}
\itemsep1pt\parskip0pt\parsep0pt
\item
  \href{https://gitter.im/JuliaLang/julia}{The JuliaLang Gitter} - A
  chatroom for Julia users
\item
  \href{https://discourse.julialang.org/}{The Julia Discourse} - A forum
  / mailing list for Julia users. Frequented by core developers.
\item
  \href{https://github.com/JuliaLang/julia}{The JuliaLang Github
  Repository} - The issues/PRs document everything changing in Julia
\item
  \href{http://www.juliabloggers.com/}{JuliaBloggers} - A blog
  aggregator for Julia
\item
  \href{https://www.reddit.com/r/Julia/}{r/Julia} - A subreddit for
  Julia
\item
  \href{http://stackoverflow.com/questions/tagged/julia-lang}{StackOverflow
  tag:julia-lang} - Q\&A site for Julia
\end{itemize}
